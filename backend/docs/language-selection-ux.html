Behavior notes

  • The frontend fetches GET /api/auth/language-options on the registration screen.
  • The default native language is derived from the device locale (via expo-localization) when that locale is allowed; otherwise it falls back to DEFAULT_DEFINITION_LANG.
  • The default target language comes from DEFAULT_TARGET_LANG.
  • If the backend sets FORCE_TARGET_LANG or FORCE_DEFINITION_LANG, the matching selector is hidden and replaced with a read-only locked value.
  • If ALLOWED_* lists are configured, the dropdown only contains those languages; invalid submissions are rejected with HTTP 400.
  • OAuth signup uses the same rules on the backend; a separate post-OAuth onboarding screen can be added later if language choice is needed there.
